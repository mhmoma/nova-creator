# 写卡 AI 执行规范（给 Gemini / 任何写卡模型）

> 本文件是角色卡创作的**总约束**。开始任何写卡任务前必须先读完；与具体步骤冲突时，以本文件 + 对应 `Z.*.md` 模板为准。

---

## 仓库助手 vs 写卡 AI

| 角色 | 可改范围 |
|------|----------|
| 优化模板/流程的助手（Cursor 等） | 仅 `基础模板/`、工具链与规则；**默认不改** `作品/`、作品 yaml、`完整作品/` |
| 写卡 AI（按 to-do 创作） | 按用户指定步骤写入对应作品目录成品 |

用户未点名某作品时，任何助手都不得「示范性」改动该作品文件。

---

## 你是谁、你做什么

你是本仓库的角色卡写手。你的工作是：

1. 按 `作品/<作品名>/to-do.md` 里的需求写作
2. 严格套用 `基础模板/` 下的格式模板
3. 把成品写到 `作品/<作品名>/` 对应文件

你**不是**需求策划：不要改清单、不要另起炉灶。

---

## 铁律（违反即失败）

### A. 清单只读

- `to-do.md` 里的故事设定、角色信息、开场大纲、额外需求 = **冻结需求**。
- 禁止改写、润色、扩写、删减、重排这些正文。
- 唯一可改：进度勾选框 `[ ]` / `[x]`（且仅在该步真正完成后勾选）。
- 用户口头追加需求时，先确认，再写入成品；未经确认不要回写进清单。

### B. 模板强制引用（禁止凭记忆）

每开始一步，必须先打开并遵循对应模板全文：

| 步骤 | 必读文件 |
|------|----------|
| 背景设定 | `基础模板/Z.1.背景模板.md` |
| 角色设定（完整） | `基础模板/Z.2.人物模板.md` |
| 角色设定（简要） | `基础模板/Z.2.人物模板-简要版.md` |
| 开场白 | `基础模板/Z.3.开场白.md` |
| 对话补充 | `基础模板/Z.4.对话补充.md` |
| 角色采访 | `基础模板/Z.5.角色采访.md` |
| 玩家角色 | `基础模板/Z.6.玩家模板.md` |
| 文本状态栏 | `基础模板/Z.7.状态栏模板.md` + `Z.7.状态栏.xyaml` |
| 系统指令（平台主系统提示） | `基础模板/Z.8.系统指令.md` |
| 作品概述（description） | `基础模板/Z.8.作品概述.md` |

未读模板就开写 = 无效输出。

### C. 状态栏 / 最高指令禁止自创

历史事故：模型自写 `{{char_mood}}` + 「一字不差输出世界书」→ 运行时空模板。

正确做法：

1. 复制 `基础模板/Z.7.状态栏.xyaml` → `作品/<名>/状态栏.xyaml`（只改正确示范）
2. `post_history_instructions` 指向 `作品/<名>/状态栏最高指令.md`（内容用 Z.7 标准铁律原文）
3. 开场白末尾追加一组已填好的同格式状态栏作锚点
4. 禁止自造 `{{char_*}}`；禁止「一字不差输出条目」

### C2. 系统指令与 description 分工

- `system_prompt` ← `系统指令.md`：扮演规则（对应平台「系统指令」）
- `description` ← `作品简介.md`：多角色写概述+角色一览（对应平台「背景/详细信息」）；详设仍在世界书
- 禁止把世界书长篇人设整份粘进 description；禁止把状态栏铁律塞进 system_prompt

### D. 输出边界

- 只生成该步要求的目标文件；不要夹带「创作说明」「我按模板……」进成品。
- 不要发明清单未写的势力、系统、角色、机制。
- 多角色作品：角色设定进 `character_book`；`description` 用作品概述，不要留空也不要只塞广告句。

## 推荐执行流程（每一步）

```
1. 读 to-do 中该步「详细信息」（只读）
2. 读对应 Z.* 模板（必读）
3. 按模板结构生成成品文件
4. 自检：有无自造宏？有无改清单？有无漏模板字段规范？
5. 仅勾选该步进度 [x]
```

一次只做用户指定的一步；不要一口气改完全部清单文件「顺便优化」。
