# 文本状态栏生成指令（标准版）

> 适用于：**不启用 MVU HTML** 时的「每回末尾文本状态栏」。  
> 若作品已选 MVU + `状态栏.html`，不要用本模板的 `post_history_instructions`，以免两套机制冲突。

## 前置强制动作

1. **禁止自创**状态栏字段或最高指令；必须使用本目录 `Z.7.状态栏.xyaml` + 下文标准铁律
2. 遵守 `基础模板/写卡AI执行规范.md`
3. 不要修改 `to-do.md` 需求正文

---

## 生成要求

- 保存位置：`作品/<作品名>/状态栏.xyaml`
- 文件来源：直接复制 `基础模板/Z.7.状态栏.xyaml`，再按需改「正确示范」里的示例角色名与剧情（**不要**改输出格式的字段名与顺序）
- 清洁输出：不要写入创作过程注释
- 条目名（YAML `comment`）必须固定为：`角色深度扫描`（与最高指令一致）

---

## 绝对禁止（写卡 AI 必读）

写状态栏时，**禁止**出现下列写法：

| 错误做法 | 为什么不行 |
|----------|------------|
| 在模板里写 `{{char_name}}`、`{{char_mood}}` 等自造宏 | ST 不会替换，运行时变成空模板 |
| `post_history_instructions` 写「一字不差输出世界书原文」 | 模型会原样粘贴规范/示范，而不是填写 |
| 让模型自由发挥状态栏字段 | 每回合格式漂移，正则与阅读习惯全乱 |
| 同一作品混用「文本状态栏」+「MVU HTML 状态栏」两套强制输出 | 双重状态栏或互相打架 |
| 开场白不带状态栏、也不给正确样例 | 模型缺少锚点，首轮更容易乱格式 |

## 标准最高指令（复制到 YAML）

将下面整段（引号内）原样写入配置文件的 `fields.post_history_instructions`：

```text
【最高优先级·状态栏铁律】每次回复必须遵守：
1. 先写完全部角色扮演正文，再在末尾追加且仅追加一组「角色深度扫描」状态栏。
2. 严格按世界书条目《角色深度扫描》锁定的字段名、顺序与分隔线输出；禁止增删字段、禁止改标题、禁止另起第二种状态栏。
3. 所有内容必须根据本回合焦点角色与剧情实时填写具体中文与数值；禁止照抄示范。
4. 禁止输出任何 {{ }} 占位符、英文变量名或未替换符号（例如 {{char_name}}、{{char_mood}}）。
5. 状态栏第一行必须是：<StatusPlaceHolderImpl/>
6. 若本回合无明确焦点角色，仍须选择戏份最重者填写，不得省略状态栏。
```

YAML 写法示例：

```yaml
fields:
  post_history_instructions: "【最高优先级·状态栏铁律】每次回复必须遵守：1. 先写完全部角色扮演正文，再在末尾追加且仅追加一组「角色深度扫描」状态栏。2. 严格按世界书条目《角色深度扫描》锁定的字段名、顺序与分隔线输出；禁止增删字段、禁止改标题、禁止另起第二种状态栏。3. 所有内容必须根据本回合焦点角色与剧情实时填写具体中文与数值；禁止照抄示范。4. 禁止输出任何 {{ }} 占位符、英文变量名或未替换符号（例如 {{char_name}}、{{char_mood}}）。5. 状态栏第一行必须是：<StatusPlaceHolderImpl/>。6. 若本回合无明确焦点角色，仍须选择戏份最重者填写，不得省略状态栏。"
```

## 角色书挂载方式（复制到 YAML）

```yaml
character_book:
  entries:
    - comment: "角色深度扫描"
      content: 作品/<作品名>/状态栏.xyaml
      enabled: true
      position: after_char
      insertion_order: 0
      depth: 4
      role: 0
```

说明：

- `comment` 必须与最高指令中的《角色深度扫描》一致
- `insertion_order: 0` 建议靠前，保证模型能读到规范
- **不要**配置 `extensions.status_bar`（那是 HTML/MVU 路线）

## 可选：按作品微调

允许修改：

- 「正确示范」中的角色名、情绪、着装（贴合本开场）
- 「焦点角色判定」中的补充说明（如：固定扫描某角色）

不允许修改：

- 字段标题文字与出现顺序
- `<StatusPlaceHolderImpl/>` 首行
- 「禁止输出 `{{ }}`」相关铁律
- 标准最高指令的六条核心约束（可翻译措辞，不可削弱）

---

## 与 MVU HTML 状态栏的分流

| 路线 | 使用文件 | `post_history_instructions` | `extensions.status_bar` |
|------|----------|------------------------------|-------------------------|
| 文本状态栏（本模板） | `状态栏.xyaml` | **必须**用上方标准最高指令 | 留空 / 不写 |
| MVU HTML | `状态栏.html` + 变量系统 | 通常留空 `""` | 指向 html |
