Nova

平台写卡清单工坊

无变量 · 对齐发布平台字段 · 输出只读 to-do

旧版 to-do-generator.html（含 MVU）保持不动，可继续单独打开。

面向发布平台流程

按栏位组织需求，一键导出创作清单

对应：作品概述、系统指令、场景、开场白、世界书人设、文本状态栏。不包含 MVU / HTML 变量状态栏。

01

基本信息

作品名将用于目录、yaml 与封面路径建议。

02

世界观 / 背景

成品：背景设定.xyaml · 模板 Z.1

03

角色设定（世界书）

成品：角色设定_姓名.xyaml · 模板 Z.2 / 简要版。详设不进 description。

04

开场白

成品：开场白.md · 模板 Z.3；若启用状态栏须在文末带填好的锚点。

05

玩家角色

成品：玩家角色_{{user}}.xyaml · 模板 Z.6

06

平台主字段

对应发布页：详细信息 / 系统指令 / 场景。多角色时 description 用作品概述，不写真人设全文。

作品概述 → description → 作品简介.md

模板 Z.8.作品概述.md。写题材、你是谁、角色一览、当前局面。

系统指令 → system_prompt → 系统指令.md

模板 Z.8.系统指令.md。只写扮演规则，不写状态栏铁律。

场景 → scenario → 场景设定.md

07

文本状态栏

无变量。复制 Z.7.状态栏.xyaml；PHI 用标准铁律文件。禁止自造宏。

将自动要求：开场白文末追加已填写状态栏锚点；post_history_instructions → 状态栏最高指令.md。

08

额外需求

叙事禁忌、尺度、必须保留的梗等。

09

预览 / 导出

生成后下载以作品名称命名的 Markdown（例如填了「逆鳞」则下载 逆鳞.md）。
